# 实训报告审查结果

## 一、格式问题

### 1. 标题样式异常（严重）

报告中所有章节标题使用了自定义 UUID 样式名（如 `21bc9c4b-6a32-...`、`71e7dc79-...`、`b63ee27f-...`），而非标准的 "Heading 1"、"Heading 2"、"Heading 3"。这会导致：

- 目录无法自动更新和正确生成
- 标题字体统一显示为宋体 12pt，缺少加粗和字号区分，视觉层次不清
- 导航窗格无法识别章节结构

**建议**：将所有一级标题（职责描述、设计描述、总结）设为 Heading 1，二级标题（项目整体概况、个人负责模块需求设计等）设为 Heading 2，三级标题（业务背景、功能需求拆解等）设为 Heading 3，并按学校模板要求设置对应字体字号（通常一级标题黑体三号/小三，二级标题黑体四号，三级标题黑体小四）。

### 2. 错别字

- 2.2.6 前端页面截图部分，"修改秘密" 应改为 **"修改密码"**。

### 3. 标题编号不规范

- "业务背景" 小节缺少编号，应为 **"2.1.1 业务背景"**。
- 功能需求拆解中使用中文序号 "（一）" 和数字 "（2）" 混用，应统一为 "（1）" 和 "（2）"，或 "（一）" 和 "（二）"。
- 功能框图部分使用 "1.1、1.2、2.1、3.1" 等编号，与文档主体编号体系脱节，建议改为与章节对应的层级编号。

### 4. 空白标题

- 2.2.5 核心功能源代码中有一个 **空的 Heading 4**（第124段），应删除或补上对应内容。

### 5. 封面信息格式不统一

- "学院名称" 一行的缩进格式与其他行不一致，且 "计算机与网络安全学院（示范性软件学院" 缺少右括号 **"）**"。
- "学生姓名"、"学生学号" 等行的首行缩进值各异，建议统一对齐。

### 6. 正文段落缩进不一致

- "项目整体概况" 下的段落有首行缩进（约2字符），而 "设计描述" 下的大部分段落没有首行缩进。
- "总结" 部分段落设为左对齐但无首行缩进，与前半部分风格不统一。
- **建议**：全文正文段落统一设置首行缩进2字符。

### 7. 正文字体未显式指定

- 大量正文段落的 run 级别没有设置字体（font=None），依赖 Normal 样式默认值。如果学校要求正文明确使用宋体小四号字，建议在样式中确认或逐段设置。

### 8. 页面边距不一致

- 第1节（封面）：左边距约2.6cm、右边距约2.4cm、上边距约3.1cm
- 第2节（正文）：左右边距均为2.54cm、上边距2.54cm
- 如果学校有统一模板要求，需检查是否符合标准（通常正文部分上下左右各2.54cm是常见要求）。

## 三、修改建议汇总

### 格式方面（优先级高→低）

1. **修复标题样式**：将所有章节标题从 UUID 样式改为标准 Heading 1/2/3/4，并按模板设置字体字号。这是最关键的问题，直接影响目录生成和文档结构。
2. **修正错别字**："修改秘密" → "修改密码"。
3. **补齐编号**："业务背景" → "2.1.1 业务背景"；统一 "（一）"/"（2）" 为一致的编号格式。
4. **删除空标题**：2.2.5 中的空白 Heading 4。
5. **修复封面**：补上学院名称的右括号。
6. **统一正文格式**：所有正文段落统一首行缩进2字符，字体统一为宋体小四。

### 内容方面

1. **补充需求规约中有但报告中遗漏的功能点**：
   - 老年人健康档案：时间轴展示、导出功能
   - 个人账户：基本信息查看/修改、邮箱绑定、头像上传
   - 如果这些功能确实未实现，应在报告中说明"未实现"或"后续迭代"，而非完全不提。

2. **对齐非功能性需求指标**：报告中写的 200ms 响应时间与需求规约的 5秒/10秒 差距很大，需要解释这个差异（例如"实际优化后达到200ms"），或统一为需求规约的指标。

3. **补充安全性描述**：需求规约要求防 XSS、CSRF 攻击和审计日志，报告中仅提到了 SQL 注入防护，建议补充或说明未覆盖的原因。

4. **细化健康档案字段描述**：需求规约中详细列出了过敏史、家族病史、生活习惯、用药情况等子项，报告中只笼统提到"基础信息+体检记录"，建议对齐需求规约的字段描述。
